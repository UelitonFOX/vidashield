Autenticação (Auth)

Login POST
/api/auth/login
Realiza autenticação de usuário via email e senha.
Pública | Retorna token JWT e informações do usuário
Registro POST
/api/auth/register
Registra um novo usuário no sistema.
Pública | Novos usuários ficam com status 'pendente'
Recuperar Senha POST
/api/auth/forgot-password
Envia email com link para redefinição de senha.
Pública | Token de redefinição com expiração de 1 hora
Redefinir Senha POST
/api/auth/reset-password
Redefine a senha do usuário com token válido.
Pública | Requer token válido enviado por email
Verificar Token GET
/api/auth/me
Verifica o token JWT atual e retorna informações do usuário logado.
Protegida | Retorna id, email, name, role, is_active e status
Logout POST
/api/auth/logout
Realiza logout do usuário no sistema.
Protegida | Requer Bearer token no cabeçalho
Token CSRF GET
/api/auth/csrf-token
Obtém um token CSRF para proteção contra ataques cross-site.
Pública | Necessário para requisições não-GET

Dashboard

Estatísticas Gerais GET
/api/dashboard/stats
Retorna estatísticas gerais para o dashboard.
Protegida | Inclui contadores de usuários, logs e alertas
Gráfico de Acessos GET
/api/dashboard/access-chart?days={7|15|30}
Retorna dados para o gráfico de acessos por período.
Protegida | Parâmetro 'days' define o período (7, 15 ou 30 dias)
Alertas Recentes GET
/api/dashboard/recent-alerts
Retorna os alertas mais recentes para o dashboard.
Protegida | Limitado aos 5 alertas mais recentes

Usuários (Users)

Listar Usuários GET
/api/users
Retorna a lista de usuários do sistema.
Protegida | Requer permissão de administrador
Detalhes do Usuário GET
/api/users/{id}
Retorna detalhes de um usuário específico.
Protegida | Usuários podem ver apenas seus próprios dados
Atualizar Usuário PUT
/api/users/{id}
Atualiza dados de um usuário específico.
Protegida | Usuários podem editar apenas seus próprios dados
Alterar Status PUT
/api/users/{id}/status
Altera o status de um usuário (ativo/inativo/pendente).
Protegida | Requer permissão de administrador
Alterar Senha PUT
/api/users/{id}/password
Altera a senha de um usuário.
Protegida | Requer senha atual para validação

Logs de Acesso

Listar Logs GET
/api/logs?page={page}&limit={limit}
Retorna logs de acesso com paginação.
Protegida | Suporta filtros por data, ação e usuário
Detalhes do Log GET
/api/logs/{id}
Retorna detalhes de um log específico.
Protegida | Inclui informações detalhadas da atividade
Exportar Logs GET
/api/logs/export?format={csv|json}
Exporta logs para CSV ou JSON.
Protegida | Requer permissão de administrador

Alertas (Alerts)

Listar Alertas GET
/api/alerts?page={page}&limit={limit}
Retorna alertas com paginação.
Protegida | Suporta filtros por severidade e status
Detalhes do Alerta GET
/api/alerts/{id}
Retorna detalhes de um alerta específico.
Protegida | Inclui informações completas do alerta
Criar Alerta POST
/api/alerts
Cria um novo alerta no sistema.
Protegida | Requer tipo e severidade
Resolver Alerta PUT
/api/alerts/{id}/resolve
Marca um alerta como resolvido.
Protegida | Requer permissão de administrador ou gerente

Configurações (Settings)

Obter Configurações GET
/api/settings
Retorna as configurações do sistema.
Protegida | Configurações sensíveis são filtradas por nível de acesso
Atualizar Configurações PUT
/api/settings
Atualiza as configurações do sistema.
Protegida | Requer permissão de administrador
Verificar Versão GET
/api/settings/version
Retorna informações sobre a versão atual do sistema.
Pública | Utilizada para verificação de compatibilidade

Informações de Segurança

Todas as rotas das APIs do VidaShield são protegidas das seguintes formas:

  • JWT (JSON Web Tokens): Para autenticação de usuários
  • CSRF Protection: Para evitar ataques Cross-Site Request Forgery
  • CORS configurado: Para permitir apenas origens confiáveis
  • Validação de dados: Todos os inputs são validados e sanitizados
  • hCaptcha: Para proteção contra bots nos formulários de autenticação
Observação Importante

Esta página de documentação é apenas para referência durante o desenvolvimento. Para ambientes de produção, é recomendado utilizar uma solução de documentação mais completa como Swagger/OpenAPI.
